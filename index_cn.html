"信任的力量与其最薄弱的环节一样弱"

LTO 网络的故事始于2014年，由文档引擎的最小可行性产品（MVP）逐渐演变为工作流程引擎。 随着时间的推移，我们的客户群变得越来越大，流程也更加全面。 作为流程的服务商，我们成为了系统用户值得信赖的第三方。

如果我们存储的数据被人为操纵了，相关组织可能会受到严重影响。 因此我们不能仅仅指望信任。 利用官僚主义或许会有所帮助，但这样会扼杀工作效率。

接着出现了区块链技术，它具有在不影响生产力的情况下同时解决所有这些问题的巨大潜力。 我们的使命是提供能够抵制官僚主义，欺诈和裙带关系的技术，用于解决困扰许多组织、行业和政府的问题。我们认为隐私和透明度并不是相互排斥的，并且我们能够证明这一点。

去中心化的工作流程自动化

每个组织机构都需要运行许多业务流程，这些流程可以定义工作流程的规则，可视化为流程图或BPMN图。 对于LTO，工作流程被表示为确定性扩展有限状态机（FSM）。

区块链使得组织机构能够以去中心化的方式运行这些工作流程，使每一方都能够利用自己的节点参与。合作方可以在链上或链下共享信息，例如文档、链接数据或（交互式）零知识证明。

这些分散的工作流程被称为交互合约 。与智能合约类似，交互合约提供一种定义区块链逻辑的动态方法。但是，交互合约的目的不只是用来判断工作状态，而是积极地指导人和计算机有关工作流程中的步骤。

微型私有链

水平可扩展性

节点可以同时保存和处理多个交互合约。由于每个交互合约都形成一个私有链，节点可以运行更多的虚拟机（VM）来处理更大的负载。

数据隐私和通用数据保护规范（GDPR）

只有特定进程参与者的节点才能访问特定的私有链。如果相关法律要求，整个工作流程及其所有相关数据都可被删除。
在此阅读我们的完整说明。

点对点通信

节点不是直接向整个网络广播事件，而是直接向对方发送。 这样可以降低复杂度和延迟。 公有的去许可链用于锚定，确保不可变的存在证明。

事件验证

每个事件都根据工作流中定义的规则和条件进行验证，所有其他参与者都将拒绝不允许的操作。

节点内部

用户界面

因为每个节点都公开JSON REST API，LTO能让您轻松地将现有系统和应用程序连接到工作进程。

事件服务

应用程序层的核心是事件服务，其处理所有传入事件，并将进一步处理委托给应用程序服务。

应用服务

应用程序层包含许多服务（如工作流程管理器），它被用于执行特定任务，还可向其添加自定义服务。

私有链的消息队列

即使在连接不良或负载较高的情况下，去耦也可确保系统的稳定性。消息队列是私有链的通信层。两个简单的服务管理所有输入和输出事件。

锚定去许可的公用链

大多数公有链主要关注智能合约和脚本化的逻辑，这使得它们对锚定等简单事务的处理缓慢、低效且过于昂贵。我们遵循类似于比特币SegWit的算法逻辑，这意味着并非所有数据都需要被所有节点永远保存。
在3秒内锚定

使用NG协议，网络每3秒预批准一次事件。如果使用其他锚定解决方案，您必须等待至多20分钟才能得到响应。

摘要块

其他公有链所需的磁盘空间往往很大，这意味着从创世块开始的同步可能需要数天时间。而LTO每天生成摘要块，之后节点可以删除单个交易。

简单的JSON REST API

每个LTO节点都有一个JSON REST API。 将锚定集成到应用程序中就像为文档或数据创建SHA哈希并通过单个HTTP请求发布一样简单。
